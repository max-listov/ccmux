# ACP — что выяснили (3 параллельных ресёрча, 2026-06-10)

## Протокол
- JSON-RPC 2.0 поверх stdio, **клиент спавнит агента** сабпроцессом. Версия 1 (integer), v2 — драфт.
- Сессии: `session/new | load (полный replay истории) | resume | list | close | delete` — 1:1 наша модель.
- Поток `session/update`: `agent_message_chunk`, `agent_thought_chunk`, `tool_call`/`tool_call_update`
  (kind read/edit/execute…, status pending/in_progress/completed/failed, **диффы**, locations),
  `plan`, `available_commands_update`, `usage_update` (токены), `config_option_update` (model/mode/effort).
- Permission flow: `session/request_permission` с опциями allow_once/always/reject — UI решает.
- Агент может звать клиента: `fs/read|write_text_file`, `terminal/*` (v2 это выкинет в MCP-over-ACP — строить через абстракцию).
- Remote (HTTP/WS): **только драфт RFD**, working group с 2026-04. stdio — единственный стабильный транспорт.
- Concurrent-доступ 2 клиентов к 1 сессии протоколом НЕ специфицирован — наша writer-проблема остаётся нашей.

## Экосистема (момент LSP)
- Управление: нейтральный орг `agentclientprotocol`, lead maintainer из **JetBrains**, со-разработка Zed+JetBrains, registry с 2026-01.
- **Native**: Gemini CLI (`--acp`), Copilot CLI, Cursor, Goose, OpenCode, Cline, OpenHands, Junie, Qwen… (~35).
- **Адаптеры Zed**: Claude (`claude-agent-acp`, бывш. claude-code-acp) и Codex (`codex-acp`, Rust). Anthropic/OpenAI native — НЕТ.
- Клиенты: Zed, JetBrains, neovim (CodeCompanion/avante), Emacs, VS Code (community), marimo (+**use-acp** React-хуки!), Obsidian, **MS Intelligent Terminal** (Build 2026).
- Headless-ниша шевелится: **acpx** (2.8k⭐ за 4 мес) — stateful headless ACP-сессии, НО single-machine, без демона/хилинга/флота.
  **«Persistent multi-machine self-healing fleet over ACP» — ниша свободна. Это ровно ccmux.**

## Claude-адаптер (`claude-agent-acp`) — критичное
- ✅ **Интероп с нашими jsonl ПОЛНЫЙ**: ACP sessionId = uuid = имя файла в `~/.claude/projects/<enc-cwd>/`.
  Резюм двусторонний (интерактивный CLI ↔ адаптер), `session/list` видит всё с диска. Условие: тот же cwd.
- Очень живой (v0.44.0 от 2026-06-09, релизы еженедельно), оборачивает **Agent SDK** (бандленный бинарь, не наш `claude`; override `CLAUDE_CODE_EXECUTABLE`).
- Полнота: permission modes 1:1 (вкл. наш auto), диффы Edit, plan/todo, thinking+effort, slash-команды (кроме интерактивных TUI: `/usage /model /statusline`), MCP passthrough, очередь промптов, AskUserQuestion-формы.
- ❌ Нет RC (Remote Control), нет statusline, субагенты не различаются визуально.
- ⚠️ **БИЛЛИНГ: с 2026-06-15 Agent SDK (=адаптер) слетает с Max-лимитов** на отдельный кредитный пул
  (Max20x = $200/мес). Интерактивный `claude` остаётся на подписке. Для постоянного флота — экономика
  меняется кардинально. Серая зона: голый `claude -p --input-format stream-json` (наш CLI, не SDK) —
  вероятно остаётся подпиской, НО надо проверить после 15.06. Community-мост через CLI:
  harukitosa/claude-code-acp («работает с Pro/Max»).

## Вердикт для ccmux
1. **ACP — «то»**: это стандартизированная форма нашей inbox-таски stream-json-driver, и она убивает
   класс багов пейн-скрейпа (idle-while-working и т.п.) по построению.
2. **Но для claude — НЕ сейчас как замена**: биллинг (15.06) делает SDK-драйв постоянного флота дорогим.
   tmux+интерактивный claude = включено в Max. Держим как основной движок claude.
3. **Гибрид — наша архитектура**:
   - Слой «источник событий» абстрагируем: jsonl-watch (claude, подписка) ИЛИ ACP-стрим (остальные агенты).
   - **Codex через codex-acp** — лучше нашей planned codex-launch таски (структурные события вместо
     второго скрейп-адаптера). Gemini/Copilot/Cursor — бесплатно падают в флот.
   - TUI/web-клиент едят ЕДИНЫЙ стрим, не знают источника.
4. **Web-клиент**: смотреть на ACP-шейп событий (или прямо ACP, когда remote-транспорт стабилизируется);
   use-acp (React-хуки) уже существует — наш WS-протокол может быть ACP-совместимым.
5. Следить: июнь 15 (биллинг, проверить голый stream-json), remote-transport RFD, v2 (fs/terminal → MCP).
